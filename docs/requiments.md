# ToDoアプリ 要件定義書 v1.0

## 1. システム概要

### システム名

ToDoアプリ

### 目的

仕事および日常のやることを管理するための、自分専用のWindowsデスクトップアプリケーションを提供する。

### 対象ユーザー

* 自分のみ

### 利用環境

* Windows
* オフライン利用

***

## 2. 技術要件

### 開発言語

* Python

### GUIフレームワーク

* PySide6

### データベース

* SQLite

### 通知

* Windowsトースト通知

***

## 3. 機能要件

### 3.1 タスク管理

#### タスク登録

以下の項目を登録できること。

| 項目   | 必須 | 備考   |
| ---- | -- | ---- |
| タイトル | 〇  | タスク名 |
| 期限   | ×  | 未設定可 |
| タグ   | ×  | 固定選択 |
| 優先度  | ×  | 5段階  |

#### タスク編集

登録済みタスクを編集できること。

#### タスク削除

登録済みタスクを削除できること。

#### タスク完了

チェックボックスによって完了状態にできること。

#### アーカイブ

完了したタスクはアーカイブへ移動すること。

#### アーカイブ復元

アーカイブ済みタスクを未完了状態へ戻せること。

***

### 3.2 タグ管理

固定タグを利用すること。

* 仕事
* 個人
* 開発
* 勉強
* 買い物
* その他

***

### 3.3 優先度管理

5段階で管理すること。

* ★★★★★
* ★★★★☆
* ★★★☆☆
* ★★☆☆☆
* ★☆☆☆☆

初期値は以下とする。

* ★★★☆☆

***

### 3.4 今日やること画面

アプリ起動時に表示すること。

以下の情報を表示すること。

#### 期限切れタスク

期限が現在日時より前の未完了タスク

#### 今日期限タスク

期限が当日の未完了タスク

#### 重要タスク

優先度が高い未完了タスク

***

### 3.5 通知機能

アプリ起動時に当日期限のタスクが存在する場合、Windows通知を表示すること。

通知内容例

```text
今日期限のタスクがあります

・会議資料作成
・顧客メール返信
```

***

***

## 5. データ要件

### タスク

| 項目名           | 型        | 備考    |
| ------------- | -------- | ----- |
| id            | INTEGER  | 主キー   |
| title         | TEXT     | タスク名  |
| due\_date     | DATE     | 期限    |
| tag           | TEXT     | タグ    |
| priority      | INTEGER  | 1～5   |
| is\_completed | BOOLEAN  | 完了フラグ |
| created\_at   | DATETIME | 作成日時  |
| completed\_at | DATETIME | 完了日時  |

***

## 6. UI要件

### デザイン方針

* シンプル
* モダン
* 視認性重視
* 操作回数を最小化

### カラーテーマ

* ダークモードを標準とする
* アクセントカラーはブルー系を採用する

***

## 7. 非機能要件

### 性能

* 数十件規模のタスク管理に対応すること
* アプリ起動から操作可能になるまで3秒以内を目標とする

### 保守性

* 機能追加しやすい構成とする
* 将来的な機能拡張を考慮した設計とする

### 信頼性

* SQLiteによる永続化を行うこと
* アプリ終了後もデータを保持すること

***
